Bug in Circuit Kit Overview

Introduction to the ANU Electronics Bug-in-Circuit IoT Kit — components, purpose and full table of contents.

🐛 What is a Bug-in-Circuit Kit?

A bug-in-circuit kit contains intentional faults — incorrect connections, missing components, wrong values or polarity mistakes — that cause circuits to malfunction. Students identify and fix these real-world bugs to build practical PCB debugging skills.

🔌

2-Channel Circuit

Two relay channels with supply and protection diode faults to debug.

⚡

4-Channel Circuit

Four relay channels with missing traces, diodes and signal path bugs.

🔧

6-Channel Circuit

Six relay channels with transistor biasing and LED indicator bugs.

ESP32 features, GPIO pin reference, communication interfaces and step-by-step Arduino IDE setup.

⚡ ESP32 Key Features
🧠

Dual-Core 240 MHz

Xtensa LX6 — one core for app logic, one for Wi-Fi/BT.

📶

Wi-Fi + Bluetooth

2.4 GHz 802.11 b/g/n Wi-Fi and BT Classic + BLE 4.2.

📌

36 GPIO Pins

Digital I/O, ADC, DAC, PWM, I2C, SPI, UART on one chip.

🔋

Low Power Modes

Deep sleep, light sleep, modem sleep for battery IoT.

📊

12-bit ADC / DAC

Analog reads on multiple channels; two 8-bit DAC outputs.

💡

16 PWM Channels

Configurable frequency and duty cycle for servos, LEDs, motors.

PCB Circuit Bug Correction

All 12 identified bugs across the 2-Channel, 4-Channel and 6-Channel relay circuits with location, problem, cause and correction.

🔌 2-Channel Circuit — 2 Bugs

Each channel: ESP32 SIG → Optocoupler (PH1U) → BC547 Transistor → Relay → Load. Both channels share 5V supply and GND.

Bug 1

Missing Supply to Optocoupler (U1H2, CH1)

🔴 Problem: PH1U4 pin 4 not receiving Vcc supply.
⚡ Cause: PCB hole connection missing.
✅ Fix: Jumper between U1H2 bottom and top holes — 5V restored to PH1U4 pin 4.
Bug 2

Flyback Diode Not Connected (Relay 2 — U5)

🔴 Problem: D2U5 not soldered across relay coil — no back-EMF protection.
⚡ Cause: Diode omission during PCB assembly.
✅ Fix: Soldered D2U5: cathode to Vcc side, anode to transistor collector.
✅
Correction process: Visual inspection → Multimeter continuity test → Channel-wise ESP32 output activation → Verify LED + relay click per channel.
⚡ 4-Channel Circuit — 4 Bugs
Bug 1

Missing Supply — Optocoupler CH1

🔴 Problem: PH1U4 pin 4 no Vcc.
⚡ Cause: PCB hole missing.
✅ Fix: Jumper U1H2 bottom → top.
Bug 2

Flyback Diode Missing (Relay 2 — U5)

🔴 Problem: D2U5 not across relay coil.
⚡ Cause: Diode omission.
✅ Fix: Soldered D2U5, correct polarity.
Bug 3

Signal Not Reaching LED (CH3 — D1U6)

🔴 Problem: D27 not connected to D1U6 anode — CH3 LED stays OFF.
⚡ Cause: PCB trace missing/misrouted.
✅ Fix: Connected D27 to D1U6 anode — LED restored.
Bug 4

Flyback Diode Missing (Relay 4 — U7)

🔴 Problem: D2U7 not across relay 4 coil.
⚡ Cause: Diode omission.
✅ Fix: Soldered D2U7 — cathode to Vcc, anode to collector.
🔧 6-Channel Circuit — 6 Bugs
Bug 1

Missing Supply — Optocoupler CH1

🔴 Problem: PH1U4 pin 4 no Vcc.
⚡ Cause: PCB hole missing.
✅ Fix: Jumper U1H2 bottom → top.
Bug 2

Flyback Diode Missing (Relay 2 — U5)

🔴 Problem: D2U5 not soldered.
⚡ Cause: Diode omission.
✅ Fix: Soldered D2U5, correct polarity.
Bug 3

Signal Not Reaching LED (CH3 — D1U6)

🔴 Problem: D27 not at D1U6 anode.
⚡ Cause: Trace missing.
✅ Fix: Connected D27 to D1U6 anode.
Bug 4

Flyback Diode Missing (Relay 4 — U7)

🔴 Problem: D2U7 missing.
⚡ Cause: Diode omission.
✅ Fix: Soldered D2U7.
Bug 5

Signal Not Reaching LED (CH5 — D1U8)

🔴 Problem: D25 not at D1U8 anode — Relay 5 LED stays OFF.
⚡ Cause: Missing PCB trace.
✅ Fix: Connected D25 to D1U8 anode — Relay 5 LED restored.
Bug 6

Transistor Base Disconnected (CH6)

🔴 Problem: BC547 base for Relay 6 not connected to base resistor — transistor never biased, relay stays OFF.
⚡ Cause: Missing trace between transistor base and resistor.
✅ Fix: Connected base to resistor — Relay 6 fully operational.
💡
Universal debug approach: (1) Visual PCB inspection, (2) Multimeter continuity on supply traces, (3) Activate each ESP32 output independently, (4) Verify LED + relay click per channel.

long getDist(){
  digitalWrite(14,LOW);delayMicroseconds(2);

💡 ESP32 LED Control via ThingSpeak
Device / PinESP32 Pin
LED (onboard/external)GPIO 2
ESP32 PowerUSB
Wi-FiRequired for cloud

ESP32 writes field1=1 to turn LED ON, then reads back the value and acts on it — full two-way cloud communication demonstrated.

C++thingspeak_led.ino
#include <WiFi.h>
#include <HTTPClient.h>
#define LED 2
const char* ssid="WIFI";
const char* pass="PASS";
String chID="CHANNEL_ID";
String wKey="WRITE_KEY";
String rKey="READ_KEY";
void loop(){
  HTTPClient h;
  // Write field1=1 (LED ON)
  h.begin("https://api.thingspeak.com/update?api_key="+wKey+"&field1=1");
  h.GET(); h.end(); delay(5000);
  // Read back field1
  h.begin("https://api.thingspeak.com/channels/"+chID+"/fields/1/last.txt?api_key="+rKey);
  if(h.GET()==200){
    String v=h.getString(); v.trim();
    digitalWrite(LED,(v=="1")?HIGH:LOW);
  }
  h.end(); delay(20000);
}

📏 4-Band vs 5-Band Resistors

🔶 4-Band Resistor (Standard)

Band 1 = 1st digit | Band 2 = 2nd digit | Band 3 = Multiplier | Band 4 = Tolerance

Value = (B1×10 + B2) × Multiplier

Example: Red-Red-Brown-Gold = (22)×10 = 220Ω ±5%

🔷 5-Band Resistor (Precision)

Band 1 = 1st | Band 2 = 2nd | Band 3 = 3rd digit | Band 4 = Multiplier | Band 5 = Tolerance

Value = (B1×100 + B2×10 + B3) × Multiplier

Example: Brown-Black-Black-Orange-Brown = 100KΩ ±1%

🥇
Reading tip: Always hold the resistor so Gold or Silver (tolerance) band is on the RIGHT, then read left to right. Memory trick: BB ROY of Great Britain Very Good Wife = 0–9 (Black Brown Red Orange Yellow Green Blue Violet Grey White).
